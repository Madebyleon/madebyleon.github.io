An Overview of the Web Development

A very very broad understanding of most things you see on a webpage can be broken down into HTML, CSS and JavaScript.

HTML represents the content and the structure of a webpage. This is the text, the images, the hyperlinks and so on.

CSS or Cascading Style Sheets represents the presentation of the content. So the font face, the colours, the background, the layout etc.

and lastly, JavaScript defines the interaction or the behaviour of the webpage. When something can be clicked or opened.

I like to imagine the relationship between these as a tree. HTML are the branches and structure of the tree, CSS represents the colour of the leaves, the size of the fruit and JavaScript is the ability to pick the fruit, shake the tree.

HTML Basics

HTML stands for Hyper Text Markup Language, which uses this idea of <tags> to describe text documents onto the browser. Our tool here is the browser itself. It deciphers the text that we write into content that can then be viewed on a browser window.

For this workshop, we will be using Google Chrome as our browser of choice.



Elements: <tags>

Elements provide the structure of the HTML page. They tell the browser how to read and interpret certain content based on what type it is. Each element takes the form of "<tags>" are how we can input content onto the browser. Most <tag> require an opening and a closing. The syntax of all <tags> are represented by:-



Some commonly used <tags> are:

  • <p> - Paragraph
  • <b> - Bold
  • <i> - Italics
  • <a href="...."> - Link to Another Page
  • <img src= "...."> - Image
  • <br> - Line Break

Attributes

All HTML elements have attributes. This is additional information that you can provide for the element. They usually come in name/value pairs.

CSS Basics

Cascading Style Sheet (CSS) is a language that describes the style of an HTML document. This is a standard for styling pages on web which include layout, design and overall presentation of web pages.

There are three different ways in which you can apply CSS properties to your HTML page.

- Inline CSS
- In header CSS
- External Stylesheet (Preferred)

Inline CSS

“Inline” CSS implies that CSS is applied directly within the HTML tag of the document, using an attribute within the tags called style.

In header CSS

In header CSS implies that the CSS is applied to the header tag in the HTML document. This is done by including a <style> tag

External Stylesheet

The most preferred way of using CSS is by putting it in an entirely new document and linking to it in your <head> tag by declaring it.

<div> tag

<div> is useful for grouping together a bunch of different HTML elements, and applying various CSS properties to them at one go.

The id attribute

id is an attribute that you can add to almost any tag in order to give them certain properties within the page. id specifies a unique id for an HTML element (the value must be unique within the HTML document). id is mostly used to point to a stylesheet, and by JavaScript (via HTML DOM) to manipulate the element with the specific id.

JavaScript Basics

JavaScript is a high-level programming language designed specifically for the World Wide Web.

Its main purpose is to bring interactivity and scripting into the browser in order to bring a diversity of behaviors in to the client-side of a webpage -- so it enables your site to respond to more sophisticated user input than just clicking on links.

Basically, JavaScript lets your user do *much more* with your page!
